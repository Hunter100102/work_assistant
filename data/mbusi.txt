TITLE: MBUSI ETA Case Handling Guide

🟩 STEP 1: Check PO in SO+

- If part has invoiced, update with invoice date and CLOSE the case.
- No further steps are needed if part has shipped.

💬 If VIN is missing in the case:
"VIN missing, please respond providing VIN in order for us to escalate accordingly."

💬 If no VIN is ever provided, close with:
"Case closed, no VIN provided by dealer for escalation."

🔍 XHPI Check:
1. Search VIN and Part Number in XHPI.
2. Confirm PN applies to VIN.
3. If mismatch, cancel order and close case.

---

🟩 STEP 2: Check STAR ORDER + LOGBUS PROD

1. Enter PO in SO+.
2. Review “Item Status.” If both lines rejected, review rejection cause (e.g., CoO).
   - Confirm if PN has CoO using “Information” tab or CMOP Excel list.
   - Notify CPM if required: cpmmbusa@daimler.com
3. If order is invoiced, add the invoice date in YBOESC case using ETD Type: “Confirmed”
4. DO NOT email dealer or leave case open unnecessarily.

---

🟩 STEP 3: Put-Away Estimate

- Ask: Do we have an estimate for "Put-away"?

---

🟩 STEP 4: Paragon & XBU Checks

🔁 ETA side uses XBU for updates and may override MBUSI Dview data.

Checklist:
1. Check YPIM:
   - Use Dealer Code 5101
   - Look for substitute parts
   - Add “ZSI added” if substituted
   - Check D2D Stock & note eligibility in case

2. Check YMEIV
3. If ATP available:
   - Add line to sales order
   - Flag “ZSI ADDED”
   - Cancel PO and close case

4. If no ATP, check for 1st SOO
5. If no SOO, move to Part E

📌 Backorder Position:
- If SOO exists, locate position in backorder queue.
- Compare total open cases vs. incoming stock in YBOESC.
- Prioritize if cases < incoming.

---

🟥 IF ZSI Case

- Change PDC & update status in YME10
- If incoming stock is elsewhere, change PDC accordingly
- Add ZSI line to Sales Order

🟥 IF ZDS Case

- Track PO that is being tracked
- Add dealer comment with updated ETA
- Create a new ETA case and link it to old one in comments

---

📦 MBUSI System Escalation – CPM MBUSI GUIDE

Website: https://sdwa0ms00001.us138.corpintra.net/

🧾 Purchase Orders:
- Use last PO Due Date + 6 weeks for ETD update
- Enter Due Date in ETD Type: "Manual"
- Document PO Due Date in internal comments

🧾 Sales Orders:
- If not visible in Open Sales Orders → check Closed
- If not visible in both → likely CMOP issue → Add to CMOP List

🚨 DO NOT escalate via XBU if:
- Supplier: SMP, Lear, Rehau, Morelli, International Auto
- PO due date is current
- Stock order
- Marked CMOP or NCOO

🆘 To escalate via XBU:
1. Click “Create MBUSI CP” in Paragon
2. Copy case ID from top-left (“XBU xxxxx”)
3. Select MBUSI Escalation as System Measure

📊 XBU Case Statuses:
- Status 1: New → Case being reviewed
- Status 2: In Process → Look for MBUSI Comments
- Status 3: Closed → Case resolved

---

⚠️ QM BLOCK

If uncommitted stock exists → email CPM:
Subject: “MBUSI Release Request + [Case Number, PN, PO]”

---

🛑 CMOP / NCOO

- CMOP: Order not received by MBUSI
- NCOO: No Country of Origin → missing contract

How to flag:
- View in Material Info > Plant Data for Material (red text)
- Save CMOP/NCOO tag in tracker

---

📦 Shipment Confirmation (ZDS/ZSI)

1. Go to DView > Sales Order Tab > Closed Sales Orders
2. Enter PO → Explode line
3. Review Shipped Date & Tracking (ZDS shows tracking number)

---

🚫 Kill Code – C01

Use if PN cannot be procured:
💬 "The part can no longer be delivered. Please make it yourself, if possible, or procure it from other sources or repair the removed part."

- Remove all data fields
- Add comment and close

---

⚠️ Invalid – Relocate to GLC

- PN may be invalid for MBUSI
- Check if extended to U.S. Market via YPIM
- Confirm order rejection
- Notify CPM to adjust parameters

Always log the state of the case with:
- Internal comment
- Your name
- Summary of manual adjustment or action

---

🔁 New Successor

- Check Old Material Number → New PN from MBUSI
- Use YPIM to confirm extension + vendor
- Use VA02 to update order
- Create new case if applicable

---

🧷 VIN-Specific Wiring Harness (C05)

- Confirm C05 via YPIM, SO+ Tab 6, or DView
- C05 = Custom built for VIN

ETD = PO Due Date + 12 to 15 weeks

Document C05 ETA logic in internal comment:
“C05 Harness | PO Due 12/3 | ETD = 15w lead = 3/18”
